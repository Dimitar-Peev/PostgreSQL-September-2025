## Database Design

### Steps in Database Design

1. Identify entities
2. Define columns
3. Define primary keys
4. Model relationships
5. Define constraints
6. Insert test data

---

### Entities

```text

Course
Town
```

## 🔗 Table Relation

---

**Table Relation** is a logical connection between tables through:

* `PRIMARY KEY`
* `FOREIGN KEY`

Goal is:

* better structure
* less duplication
* easier JOIN queries
* guaranteed data consistency

---

## 🔑 Primary & Foreign Keys

---

### Primary Key Rules

* Every table should have a primary key
* Usually:

    * Integer
    * Unique
    * First column in the table

---

### Primary Key Implementations

```sql id="t9y2m1"
"id" INT PRIMARY KEY

"id" SERIAL PRIMARY KEY -- Auto-increment

"id" INT GENERATED ALWAYS AS IDENTITY -- Auto-increment always (cannot provide an explicit value)

"id" INT GENERATED BY DEFAULT AS IDENTITY -- Auto-increment (cannot guarantee uniqueness)
```

---

### Foreign Keys

* Foreign keys connect related tables
* Prevent duplicated data

---

### Basic Foreign Key

```sql id="0nm2yw"
CREATE TABLE "clients"
(
    "id"   SERIAL PRIMARY KEY,
    "name" VARCHAR(30)
);

CREATE TABLE "orders"
(
    "id"        SERIAL PRIMARY KEY,
    "client_id" INT REFERENCES "clients"
);
```

---

### Named Foreign Key Constraint

```sql id="sjh8i5"
CREATE TABLE "orders"
(
    "id"        INT PRIMARY KEY,
    "client_id" INT,

    CONSTRAINT fk_client
        FOREIGN KEY ("client_id")
            REFERENCES "clients" ("id")
);
```

## 🔗 Relationship Types

---

### One-to-Many

Example:

```text
Mountain → Peaks
```

One mountain can have many peaks.

---

### One-to-Many Setup

```sql id="4tdf9j"
CREATE TABLE "mountains"
(
    "id"   SERIAL PRIMARY KEY,
    "name" VARCHAR(50)
);

CREATE TABLE peaks
(
    "id"          SERIAL PRIMARY KEY,
    "name"        VARCHAR(50),
    "mountain_id" INT REFERENCES "mountains"
);
```

---

### Many-to-Many

Example:

```text
Employees ↔ Projects
```

Requires a mapping table.

---

### Many-to-Many Setup

```sql id="m5uvy5"
CREATE TABLE "employees"
(
    "id"            SERIAL PRIMARY KEY NOT NULL,
    "employee_name" VARCHAR(50)        NOT NULL
);

CREATE TABLE "projects"
(
    "id"           SERIAL PRIMARY KEY NOT NULL,
    "project_name" VARCHAR(50)        NOT NULL
);

-- MAPPING TABLE
CREATE TABLE "employees_projects"
(
    "employee_id" INT,
    "project_id"  INT,

    -- COMPOSITE PRIMARY KEY
    CONSTRAINT pk_employees_projects
        PRIMARY KEY ("employee_id", "project_id"),

    CONSTRAINT fk_employees_projects_employees
        FOREIGN KEY ("employee_id")
            REFERENCES "employees" ("id"),

    CONSTRAINT fk_employees_projects_projects
        FOREIGN KEY ("project_id")
            REFERENCES "projects" ("id")
);
```

---

### One-to-One

Example:

```text
Country ↔ Capital
```

---

### One-to-One Setup

```sql id="0slf2q"
CREATE TABLE "capitals"
(
    "capital_id"   SERIAL PRIMARY KEY,
    "capital_name" VARCHAR(50)
);

CREATE TABLE "countries"
(
    "country_id" SERIAL PRIMARY KEY,
    "capital_id" INT UNIQUE,

    CONSTRAINT fk_countries_capitals
        FOREIGN KEY ("capital_id")
            REFERENCES "capitals" ("capital_id")
);
```

## 🔄 JOINs

* JOINs combine data from multiple tables

---

### Basic JOIN Syntax

```sql id="i5y0db"
SELECT *
FROM table_a
         JOIN table_b
              ON table_a.id = table_b.id;
```

---

### Example: Drivers & Vehicles

```sql id="ehy4vc"
SELECT "driver_id",
       "vehicle_type",
       CONCAT("first_name", ' ', "last_name") AS "driver_name"
FROM "vehicles" AS v
         JOIN "campers" AS c
              ON v."driver_id" = "c".id;
```

---

### Example: Hiking Routes

```sql id="5h3j8f"
SELECT "start_point",
       "end_point",
       "leader_id",
       CONCAT(c."first_name", ' ', c."last_name") AS "leader_name"
FROM "routes" AS r
         JOIN "campers" AS c
              ON r."leader_id" = c."id";
```

## 🧨 Cascade Operations

* CASCADE propagates changes to related records

---

### ON DELETE CASCADE

Deletes related child records automatically.

```sql id="h8j6lc"
CREATE TABLE "drivers"
(
    "driver_id"   INT PRIMARY KEY,
    "driver_name" VARCHAR(50)
);

CREATE TABLE "cars"
(
    "car_id"    INT PRIMARY KEY,
    "driver_id" INT,

    CONSTRAINT fk_car_driver
        FOREIGN KEY ("driver_id")
            REFERENCES "drivers" ("driver_id")
            ON DELETE CASCADE
);
```

---

### Example: Mountains & Peaks

```sql id="n6mxvb"
CREATE TABLE "mountains"
(
    "id"   SERIAL PRIMARY KEY,
    "name" VARCHAR(50) NOT NULL
);

CREATE TABLE "peaks"
(
    "id"          SERIAL PRIMARY KEY,
    "name"        VARCHAR(50) NOT NULL,
    "mountain_id" INT,

    CONSTRAINT fk_mountain_id
        FOREIGN KEY ("mountain_id")
            REFERENCES "mountains" ("id")
            ON DELETE CASCADE
);
```

---

### ON UPDATE CASCADE

Updates foreign keys automatically.

```sql id="z4k2xy"
CREATE TABLE cars
(
    "car_id"    INT PRIMARY KEY,
    "driver_id" INT,

    CONSTRAINT fk_car_driver
        FOREIGN KEY ("driver_id")
            REFERENCES "drivers" ("driver_id")
            ON UPDATE CASCADE
);
```

## ⚙️ Common Patterns

---

### Create Parent + Child Tables

```sql id="wxpfj9"
CREATE TABLE "authors"
(
    "id"   SERIAL PRIMARY KEY,
    "name" VARCHAR(50)
);

CREATE TABLE "books"
(
    "id"        SERIAL PRIMARY KEY,
    "title"     VARCHAR(100),
    "author_id" INT REFERENCES "authors" ("id") -- Short way without name

--  CONSTRAINT fk_books_authors -- Longer way with constraint name
--      FOREIGN KEY ("author_id")
--          REFERENCES "authors" ("id")
);
```

---

### Join Two Related Tables

```sql id="z6r4rj"
SELECT b."title",
       a."name"
FROM "books" AS b
         JOIN "authors" AS a
              ON b."author_id" = a."id";
```

---

### Create Mapping Table

```sql id="v0w0k8"
CREATE TABLE "students_courses"
(
    "student_id" INT,
    "course_id"  INT,

    PRIMARY KEY ("student_id", "course_id")
);
```

## ✅ Summary

* Database design starts with entities and relationships
* Primary keys uniquely identify rows
* Foreign keys connect related tables
* Main relationship types:

    * One-to-one
    * One-to-many
    * Many-to-many
* JOINs retrieve data across multiple tables
* CASCADE operations automate updates and deletions
* E/R diagrams visualize database structure
